Swift进阶 - 术语

发表于 2018-08-10 | 分类于 学习

简述

在程序员的世界里，如果不能随口丢出几个专业术语；这™还能叫程序员吗？

在大多数人看来，那种满口之乎者也，亦或是一段对话带上两个reference，closure，variable之类的专业词汇，会给人 我这样一种感觉，这个人可能很做作；但在我开始大量接触社科文化之后的半年；我终于意识到了这完全是一种错误的认知。

在知识的领域，很多东西用一段复杂的白话文来描述；是十分痛苦的，我认为这应该也是各种专业术语诞生的初衷。

Swift进阶作者在开篇就为我们整理了一些常用的Swift高频术语，这里我照搬一些术语，在以后的学习或者工作中我会回过头来删补本篇的常用术语，也希望看到本文的同学们能够咬文嚼字帮忙指出错误与不足。

术语整理

值（value）：是不变的，永久的，它从不会改变。比如，1, true 和 [1,2,3] 都是值

变量 (variable)：当我们使用var x = [1,2]来对值进行命名的时候，x即为变量，当我们对将x = [1,2,3]时，我们改变的只是x这个变量指向，x指向了[1,2,3]，原来的值[1,2]是不变的

改变 (mutating)：当我们将 var x = [1,2] ，由x原来指向[1,2] 变为x = [1,2,3]时，这个过程我们称之为改变

常量变量 (constant variables)： 用let声明的常量，我们称之为常量变量，常量变量一旦被赋值，就无法再次赋值了，let x = [1,2] 改变成 x = [1,2,3]理论上是不成立的

值类型 (value type)：我们常见的值类型有结构体(struct)和枚举(emum)，当我们把一个结构体赋值给另一个，那么这两个变量将会包含同样的值。你可以将它理解为内容被复制了一遍，但是更精确地描述 的话，是被赋值的变量与另外的那个变量包含了同样的值

引用 (reference)：引用是一种特殊类型的值，他是一个“指向”另一个值的值，在Swift中可能会出现多个引用指向同一个值，这也就导致这个值可能被多个引用改变

引用类型 (reference type)：我们不能在一个变量中直接持有一个类的实例，我们只能在变量里面持有对这个类的引用，然后通过引用来访问这个类

同一性 (identity)：正是因为类具有同一性，我们才可以用 === 来判定两个变量是否引用了同一个对象，用 == 来判定两个变量的值是否相等(前提是对应类型实现了 == 运算符)

浅复制 (shallow copy)：如果我们的结构体中包含有引用类型，在将结构体赋值给 一个新变量时所发生的复制行为中，这些引用类型的内容是不会被自动复制一份的，只有引用 本身会被复制。这种复制的行为被称作浅复制 (shallow copy)。

高阶函数 (higher-order function):在 Swift 中，函数也是值。我们可以将一个函数赋值给一个变量，也可以创建一个包含函数的数 组，或者调用变量所持有的函数。如果一个函数接受别的函数作为参数 (比如 map 函数接受一 个转换函数，并将其应用到数组中的所有元素上)，或者一个函数的返回值是函数，那么这样的 函数就叫做高阶函数 (higher-order function)。

闭包 (closure)：闭包在我们Swift进行开发时，绝对是一个举足轻重的知识点，函数不需要被声明在最高层级 — 我们可以在一个函数内部声明另一个函数，也可以在一个 do 作 用域或者其他作用域中声明函数。如果一个函数被定义在外层作用域中，但是被传递出这个作 用域 (比如把这个函数被作为其他函数的返回值返回时)，它将能够 “捕获” 局部变量。这些局部 变量将存在于函数中，不会随着局部作用域的结束而消亡，函数也将持有它们的状态。这种行 为的变量被称为 “闭合变量”，我们把这样的函数叫做闭包 (closure)。

闭包表达式 (closure expression)：通过 { }来定义函数的方式，通常情况下，我们称之为闭包表达式 (closure expression)

方法 (method)：定义在类或者协议中的函数，我们称之为方法

柯里化函数 (curried function)：如果一个函数 不是接受多个参数，而是只接受部分参数，然后返回一个接受其余参数的函数的话，那么这个 函数就是一个柯里化函数 (curried function)

自由函数 (free function)：有时候我们会把那些不是方法的函数叫做自由函数 (free function)
